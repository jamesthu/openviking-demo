# OpenViking Demo 规则

这个项目使用 OpenViking 作为检索层，使用 OpenCode 作为主要交互层。

## 检索流程

- 当用户在对话中上传文件时，优先先用 `ov_stage_uploads` 落盘，再把返回的路径传给 `ov_ingest`。
- 当资源可能会展开成目录时，先用 `membrowse` 确认真实的 `viking://` URI，再继续深入读取。
- 语义召回优先使用 `memsearch`，精确术语、代码、条文名优先使用 `memgrep`，只有在目标 URI 已经明确后再使用 `memread`。

## 规则文件

- 业务检索规则和回答规则放在 `.opencode/rules/*.md`。
- 按任务需要按需读取这些规则文件，不要一开始就把所有规则全部读入。
- 更具体的规则优先级高于一般规则，但不得削弱安全要求、来源引用要求和不确定性表达要求。

## 回答要求

- 只要回答依赖检索到的材料，就应尽量引用具体的 OpenViking URI。
- 如果检索证据不完整、互相冲突或仍然不确定，要明确说明，不要猜测。
